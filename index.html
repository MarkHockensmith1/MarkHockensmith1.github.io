Welcome To the project portfolio for my seinor project: Task Managment

Mark Hockensmith


The design of the application will be moble based. The user will install a mobile application onto their smart phone and use it to create, update and check tasks. The user will be notified about upcoming tasks via notifications on the phone and through the user detection system.

Three Components

Mobile Application

The Interface of the application runs on a mobile phone, where users input tasks to be completed, edit tasks, and recieve notifications on the status of a Task

Backend Cloud Application

Stores data from the mobile application and is responsible for any logic and sending notifications to the app or the Pi

Pi Notification System

Detects a user entering an office or similar loacation and notifies them of tasks due that day

Mobile Application

The Mobile application is Built using Android Studio and Java. All user interaction is on the mobile application, Users register, enter, view, edit, and recieve notifications on tasks. The application connects to the Cloud application through Json. The mobile application is meant to be very quick and simple so that a user can quickly jot input a task and continue whatever they are doing with minor interuption

Backend Cloud Application

The Cloud application is built  using php and the Laravel MVC framework.  Rest Controllers take JSON get and post requests from the mobile application and notification system and performs logic on the requests as well as connects to the database to create, update, read, and delete users and tasks.

This method is responsible for creating a new task in the database and returning the created task back to the business layer. it does this by taking in an instance of a task model created in the controller from the json input. Notification System

Due to time consraints the Notification system was moved out of scope for this class. The system consisted of a raspberry pi with a motion sensor and speaker attached. When the pi would detect a user had entered a space, a request to the cloud application would check for upcoming due tasks, they would be sent back to the pi, and it would use a speech program to inform the user.
